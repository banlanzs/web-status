# 修复记录：暂停监控状态显示 (Paused Status Fix)

**问题**: 当监控处于暂停状态 (Paused) 时，跨天的状态条错误地显示为 100% 可用（绿色），而非无数据/暂停（灰色）。

**修复内容**:

1.  **逻辑重构 (Log Processing)**
    *   **原逻辑**: 仅匹配日志的`开始日期`。
    *   **新逻辑**: 计算日志时间段与当天的**重叠 (Overlap)**。无论暂停持续几天，都能正确识别每一天的暂停时长。

2.  **可用率计算 (Uptime Calculation)**
    *   从分母中剔除暂停时间：`可用率 = (总时间 - 暂停时长 - 故障时长) / (总时间 - 暂停时长)`。
    *   全天暂停时，可用率不再显示为 100%，而是特定状态。

3.  **UI 状态优先级 (Status & Color)**
    *   新增判定：若 `暂停时长 ≈ 全天` 或 `当前状态 == paused`，强制显示为 **灰色 (bg-slate-400)**。
    *   优先级调整：未来日期 > **暂停** > 故障 > 正常。

4.  **交互优化 (Tooltip)**
    *   暂停状态下，Tooltip 不再显示 "可用率: 100%"，而是直接显示 **"已暂停"**。
